# Production Environment Config Checklist

For the v1.0 Launch, the following environment variables MUST be configured exactly as described.
**WARNING: NEVER COMMIT REAL PRODUCTION VALUES to `.env` or `.env.local`!** They should only be configured locally on the build machine or in secure CI/CD pipelines. `.env.example` must contain placeholders only.

## 1. Application State
| Variable | Required? | Safe Example Placeholder | Production Location |
| :--- | :--- | :--- | :--- |
| `VITE_APP_VERSION` | YES | `1.0.0` | Local build machine only |
| `VITE_ENCRYPTION_SALT` | YES | `YOUR_SECURE_SALT_HERE` | Local build machine only |

## 2. Feature Flags (Staged Beta / Release Gates)
| Variable | Required? | Production Value |
| :--- | :--- | :--- |
| `VITE_ENABLE_MULTIPLAYER` | YES | `true` (if deploying to beta) otherwise `false` |
| `VITE_ENABLE_RUST_REALTIME` | YES | `true` (if deploying to beta) otherwise `false` |
| `VITE_ENABLE_RANKED_ARENA` | YES | `true` (if deploying to beta) otherwise `false` |
| `VITE_ENABLE_CHALLENGE_MATCH` | YES | `true` (if deploying to beta) otherwise `false` |
| `VITE_ENABLE_TOURNAMENTS` | YES | `true` (if deploying to beta) otherwise `false` |

## 4. Backend Cryptographic Secrets (BACKEND ONLY)
| Variable | Required? | Purpose | Safe Example Placeholder | Location |
| :--- | :--- | :--- | :--- | :--- |
| `SESSION_TOKEN_SECRET` | YES | Signs short-lived HMAC user session tokens verified by Rust backend | `a_very_long_random_session_key_secret_32_bytes_min` | Hosting Env settings only |
| `RANKED_RESULT_HMAC_SECRET` | YES | Signs ranked match ELO results generated by Rust and verified by Node | `another_different_long_random_ranked_hmac_secret` | Hosting Env settings only |
| `GEMINI_API_KEY` | YES | Backend Gemini model narration key | `AIzaSy-your-key-here` | Hosting Env settings only |

### Critical Rules:
- **Separation of Secrets**: `SESSION_TOKEN_SECRET` and `RANKED_RESULT_HMAC_SECRET` MUST use completely different, strong random strings.
- **Backend Only**: Never expose `SESSION_TOKEN_SECRET` or `RANKED_RESULT_HMAC_SECRET` in frontend files or Web Console variables.
- **No Localhost**: Ensure no `ws://localhost` or `http://localhost` references are left in production paths.
- **No Committed Secrets**: Never commit `.env` or service accounts.
