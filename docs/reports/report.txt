# Clash of Crowns - Technical Report & System Overview

## 1. System Architecture: Server vs. Client Data

### Client-Side (User Side)
The application is a high-performance React Single Page Application (SPA). Most of the heavy lifting, including 3D rendering and AI move generation, happens directly on the user's device to ensure low latency.
- **Game State**: Current board position, move history, and captured pieces are managed locally in the React state.
- **3D Rendering**: Handled by Three.js (via React Three Fiber), utilizing the user's GPU.
- **AI Engines**: 
  - **Internal Engine**: A custom Negamax-based engine running in the main thread for instant responses at lower levels.
  - **Stockfish WASM**: A full Stockfish engine running in a Web Worker for high-level play.
- **Local Storage**: Used to persist game state between sessions (Resume feature).

### Server-Side (Firebase)
- **Authentication**: Firebase Auth manages user identities and secure login.
- **User Profiles**: Firestore stores persistent user data:
  - Elo Rating
  - Win/Loss/Draw statistics
  - Unlocked Tiers and Characters (Progress)
  - Customization preferences (Piece sets, board themes, home animations)
- **Leaderboard**: Global rankings are calculated and served from Firestore.

## 3. The AI Brain: Logic & Depth Levels

The AI system uses a hybrid approach to balance speed and difficulty.

### Internal Engine (Tiers 0-1)
For the first two tiers, the game uses a custom-built engine optimized for **instant response**.
- **Algorithm**: Negamax with Alpha-Beta Pruning.
- **Optimizations**: Transposition Tables (Zobrist Hashing), Move Ordering (MVV-LVA), Principal Variation Search (PVS), and Iterative Deepening.
- **Response Time**: Near-instant (target < 100ms).

### Stockfish WASM (Tiers 2-5)
For intermediate and master levels, the game utilizes Stockfish for professional-grade play.
- **Communication**: Asynchronous Web Worker to prevent UI freezing.
- **Skill Levels**: Dynamically adjusted (0-20) based on the specific character.
