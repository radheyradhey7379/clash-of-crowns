# NNUE Dataset Strategy

This document outlines the strictly legally safe dataset collection methodology for Clash of Crowns' NNUE v2.0.

## Core Rules
1. **No GPL Datasets**: We will not scrape or download open-source Stockfish datasets, `.binpack` files, or proprietary network weights.
2. **Self-Play Generation**: Our primary dataset source will be self-play games generated by the internal Rust engine (HCE vs HCE, NNUE-placeholder vs NNUE-placeholder).
3. **No Third-Party PGNs**: Unless explicitly licensed or public domain, third-party PGN files will not be included in the training set.
4. **Benchmark Labels**: Stockfish may only be used locally via command-line tooling to generate highly accurate benchmark labels for validation, but NEVER shipped as a gameplay engine.

## Repository Hygiene
The repository must remain lean. Large datasets will bloat git history.
The `.gitignore` has been updated to exclude:
- `data/nnue/`
- `*.nnue`
- `*.pt`, `*.pth`, `*.ckpt`
- `*.bin.tmp`

*Exception: Tiny mock files explicitly built into `tools/nnue_dataset` for CI/CD testing are permitted.*
